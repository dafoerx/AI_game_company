# 主题 / 平台 / 商业目标定稿

## 1. 最终定稿

### 项目主题
**餐饮门店经营模拟**

一句话描述：

> 玩家经营一家小型餐饮门店，在菜单选择、备料、人手安排、出餐效率、顾客满意度和现金流之间做取舍，并逐步把门店经营稳定、做大。

### 目标平台
**Web**

### 商业目标
**原型验证**

---

## 2. 题材筛选结论

当前阶段优先看的是：

- 是否适合小团队快速实现
- 是否适合浏览器表达
- 是否容易验证“继续经营下去”的动力
- 是否能在 15～30 分钟内形成反馈循环

### 备选题材对比

| 题材 | 优点 | 问题 | 结论 |
|---|---|---|---|
| 农场经营 | 易理解，受众广 | 生长周期偏长，反馈点容易变慢 | 可参考 |
| 游戏公司经营 | 题材辨识度高，系统感强 | 前置解释成本更高，文档链条更容易变重 | 暂缓 |
| **餐饮门店经营** | 接单、出餐、翻台、口碑、现金流都直观；短循环反馈明显；很适合 Web 面板表达 | 需要控制品类与动线复杂度 | **首选** |

---

## 3. 为什么 Web 是对的

对于“原型验证”目标，Web 仍然最合适：

1. **开发速度快**：优先做状态和流程，不必重投入表现层。
2. **信息组织自然**：订单、库存、顾客反馈、设备状态都适合网页界面。
3. **试玩门槛低**：内部试玩和小范围验证都方便。
4. **便于快速改数值和布局**：适合餐饮经营这类需要反复调节节奏的原型。

---

## 4. 为什么商业目标先定为“原型验证”

当前阶段不追求直接商业化，而是优先回答：

- 这个题材是否足够直观且有扩张动力？
- 玩家是否能理解备料、出餐、满意度和现金流的取舍？
- 营业节奏是否足够紧凑，能支撑 15～30 分钟体验？
- Web 形态是否足够承载核心经营乐趣？

只有这些问题得到验证，后面再考虑扩展内容、正式立项和商业化路径。

---

## 5. 首版核心幻想与资源建议

### 玩家幻想
- 我能把一家小店经营起来
- 我能通过菜单和设备升级提升效率
- 我能平衡高峰期压力与顾客满意度
- 我能靠稳定经营赚到钱并逐步扩张

### 首版核心资源建议
- **现金**：决定备料、雇人、升级设备
- **库存**：决定能否持续出餐
- **员工精力 / 人手**：决定制作与服务效率
- **顾客满意度**：决定评价、复购和口碑
- **营业进度 / 当日时段**：决定营业节奏与高峰压力

---

## 6. 对应的首版核心循环

**开店准备 → 接单与分配制作 → 出餐与服务 → 获取收入与评价 → 处理库存与损耗 → 升级菜单 / 设备 / 人手 → 进入下一轮营业**

这个循环足以支撑一个 Web 原型版本的首轮验证。
