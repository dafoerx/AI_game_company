# MVP 范围规划（餐饮项目版）

## 1. MVP 目标

首个版本不追求“餐厅内容很多”，而是优先验证以下问题：

1. 玩家是否能快速理解营业日经营闭环
2. 玩家是否愿意持续进行下一轮营业
3. 菜单、库存、人手和满意度之间是否形成策略取舍
4. Web 形态是否足够承载餐饮经营的核心乐趣

## 2. 建议的最小可玩闭环

建议先控制在以下循环：

**营业前准备 → 接单 → 制作 / 出餐 → 获得收入与评价 → 处理库存与人手压力 → 升级 / 调整菜单 → 下一轮营业**

## 3. MVP 必做系统

### 核心系统
- 基础营业日循环
- 订单与顾客需求系统
- 菜单与出餐系统
- 库存 / 备料系统
- 基础人手 / 工位分配
- 收入、成本与现金流系统
- 顾客满意度 / 评价反馈
- 基础升级与解锁系统
- 存档与读取
- 核心 UI 流程

### 体验保障
- 基础引导
- 关键反馈（出餐成功、差评、售罄、营业结束）
- 基础数值平衡

## 4. MVP 暂不建议做的内容

- 多店并行经营
- 复杂装修自由度
- 复杂剧情与角色支线
- 复杂联网功能
- 大量随机事件
- 复杂外卖 / 堂食双系统并行
- 深度供应链模拟
- 完整营销活动系统

## 5. MVP 完成标准

至少满足以下条件：

- 玩家能够从开店准备到完成一轮营业并结算
- 玩家能理解主要资源如何获取和消耗
- 系统不会在中途明显失衡或失去目标
- 玩家在 15～30 分钟内能感受到继续经营和优化的动机

## 6. 后续扩展方向

当 MVP 验证成立后，再考虑补充：

- 更多菜品和顾客类型
- 更多设备与工位组合
- 连锁扩张与分店经营
- 活动营销和品牌经营
- 剧情包装与更完整世界观
