# MVP 范围规划

## 1. MVP 目标

首个版本不追求“内容很多”，而是验证以下问题：

1. 玩家是否能快速理解核心循环
2. 玩家是否愿意持续进行经营扩张
3. 系统之间是否能形成基本正反馈
4. 游戏节奏是否具备继续扩展的基础

## 2. 建议的最小可玩闭环

建议先控制在以下循环：

**建设/配置 → 生产/服务 → 获得收益 → 升级/扩张 → 解锁新内容**

## 3. MVP 必做系统

### 核心系统
- 基础经营循环
- 资源与货币系统
- 建筑/设施或功能位系统
- 顾客/订单/需求系统
- 升级与解锁系统
- 基础任务/目标系统
- 存档与读取
- 核心 UI 流程

### 体验保障
- 基础引导
- 关键反馈（收益、升级、完成）
- 基础数值平衡

## 4. MVP 暂不建议做的内容

在主题未完全明确前，以下内容建议暂缓：

- 复杂剧情系统
- 过深社交系统
- 复杂联网功能
- 大量角色故事线
- 多层科技树
- 复杂装修自由度
- 大量随机事件
- 多地图并行经营

## 5. MVP 完成标准

至少满足以下条件：

- 玩家能够从 0 到 1 跑通一次完整经营成长流程
- 玩家能理解主要资源如何获取和消耗
- 系统不会在中途明显失衡或失去目标
- 玩家在 15～30 分钟内能感受到持续扩张的动机

## 6. 后续扩展方向

当 MVP 验证成立后，再考虑补充：

- 更多经营模块
- 更多顾客行为层次
- 更多成长与策略系统
- 剧情包装与世界观
- 活动、成就或弱联网内容
