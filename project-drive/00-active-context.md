# 00-active-context

**最后更新**：2026-03-29
**项目名称**：中转之家：放心去爱（Web原型）
**当前阶段**：项目启动与规划
**视觉风格**：温暖绘本感 2D 手绘风格，低饱和治愈配色；动物表情细腻分层（警惕、放松、亲近），界面采用轻量卡片化信息结构；关键情绪节点使用照片墙与短视频拼贴式过场，强化“陪伴轨迹”与“送别时刻”。
**目标平台**：Web（PC 浏览器优先，适配移动端浏览器）

---

## 1. 项目主题

玩家作为流浪动物中转之家的经营者，在有限时间与资源中照料并理解受过创伤的动物，通过观察、陪伴与互动重建它们对人和家庭的信任，最终做出“何时放手”的领养决定；项目情感核心是治愈与被治愈、记录与告别、以及温柔地让爱流动。

## 2. 核心循环

玩家每天接收新动物并读取其背景线索，选择观察、陪伴、互动小游戏等干预行动来降低应激、提升信任并逐步解锁“心结故事”；当动物达到领养准备阈值后，玩家为其制作社交发布内容并匹配领养家庭，完成送养后收到回访照片/短视频反馈，再进入下一只动物的接收与照护循环。

## 3. MVP 范围

### 必做
- 动物入站系统（至少3类创伤原型）
- 双核心状态：信任度/应激度（含每日变化与行动反馈）
- 三类基础行动：观察、陪伴、互动小游戏
- 心结解锁事件（每只动物至少2段关键剧情）
- 领养家庭匹配系统（偏好与禁忌规则）
- 送养流程与回访消息（照片卡片+文字）
- 基础经营约束（每日行动点与简化预算）

### 可延后
- 更多动物种类与个性动画
- 季节主题活动与节日回访
- 短视频模板可自定义滤镜/贴纸
- 更复杂的家庭长期关系追踪

### 明确不做
- 多人联机与玩家社交系统
- 真实第三方社交媒体 API 对接
- 3D 场景与高成本骨骼动画
- 付费商业化与商城系统
- 开放式地图探索玩法

### M3：测试迭代与Demo验收
- **周期**：2周

- **主责**：prototype-qa

## 7. 风险

- 情感题材处理不当可能造成玩家心理负担，削弱治愈体验
- 剧情与动物事件内容生产成本高，易成为进度瓶颈
- Web端在低配移动浏览器上的性能与适配风险
- 核心循环若反馈过慢，玩家可能在首轮体验中流失
- 团队对“克制MVP”执行不严格导致范围膨胀

## 8. 成功标准

在8周内交付可稳定演示的Web MVP；新玩家5分钟内理解玩法、15分钟内完成一次完整闭环；测试用户中至少70%愿意继续体验下一只动物，且至少80%对“治愈与放手”的情绪体验给出正向反馈；PC端运行流畅且移动端核心流程可用。
