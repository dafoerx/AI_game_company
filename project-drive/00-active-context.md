# 00-active-context

**最后更新**：2026-03-28
**项目名称**：《中转之家：放心去爱》
**当前阶段**：项目启动与规划
**视觉风格**：温暖绘本感 2D 手绘风格，低饱和治愈配色；动物表情演出细腻，界面偏轻量卡片化，过场采用照片墙与短视频拼贴，突出“记录与告别”的情绪体验。
**目标平台**：Web（PC 浏览器优先，适配移动端浏览器）

---

## 1. 项目主题

在 Web 端经营一家流浪动物中转之家，玩家通过观察行为、陪伴互动与情绪判断，逐步修复动物的创伤与信任问题，再借助社交发布为它们匹配合适家庭；在产生深厚感情后学会“适时放手”，并通过回访见证它们的新生活，传递治愈与被治愈的温柔内核。

## 2. 核心循环

玩家每日接收流浪动物并查看其背景档案，先通过观察行为与互动反馈识别“心结”，再安排陪伴与小游戏提升信任、降低应激；当动物达到可领养状态后，制作并发布征婚视频卡片，匹配潜在家庭并做送养决策；送养后获得回访照片/视频，解锁新事件与新救助个体，进入下一轮“接收-疗愈-放手”的循环。

## 3. MVP 范围

### 必做
- 动物接收与基础档案系统（首版 3 类动物：狗、猫、兔）
- 心结-信任-应激三维状态系统与可视化反馈
- 2 种核心陪伴互动（观察对话 + 轻量小游戏）
- 站内社交发布功能（征婚视频卡片生成与发布）
- 领养家庭匹配与送养判断流程（含成功率提示）
- 送养后回访事件（照片/短文字）

### 可延后
- 更多动物品种与个体化剧情分支
- 志愿者 NPC 关系线
- 节日主题活动与限时回访
- 轻成就与收藏相册系统

### 明确不做
- 多人联机与社交公会
- 复杂 3D 场景与自由探索
- 真实货币支付与商业化广告系统
- 高复杂度装修建造玩法
- 完整语音配音系统

## 7. 风险

- 情绪主题过重导致部分玩家体验压力大，影响留存
- 动物事件内容量不足会削弱重复游玩价值
- 匹配系统若反馈不透明，玩家会误解为“随机失败”
- Web 端性能与资源加载不稳定影响首屏与交互流畅度
- 团队在原型期过度扩张功能，导致核心闭环无法按期验证

## 8. 成功标准

7周内交付可在主流浏览器稳定运行的MVP，玩家可完整体验“接收-疗愈-发布-匹配-送养-回访”闭环；首轮测试中至少70%玩家能明确感知“放手也是爱”的主题，至少60%玩家表达二次游玩意愿，且核心流程无阻断级缺陷。
