# producer role board

**角色**：制作人 / 主策划  
**当前阶段定位**：范围裁决者、优先级拍板者、任务收口负责人

## 开始前必读

1. `project-drive/00-active-context.md`
2. `project-drive/02-task-cards/in-progress/TASK-001_核心循环与资源流及首页信息架构.md`

## 当前轮到你时要做什么

当技术、UI、QA 的评审意见回收后，你要做的不是继续发散，而是：

- 判断 `TASK-001` 是否仍然符合 MVP 边界
- 删除超出首版承载能力的复杂设计
- 明确“现在必须做 / 可以后做 / 现在不做”的边界
- 决定下一阶段由谁先落地实现

## 当前确认项

- 5 个核心资源是否已经是首版上限
- 15～30 分钟体验节奏是否合理
- 主界面信息量是否仍在 Web 原型可承载范围内
- 是否允许并行项目、随机事件、员工疲劳等机制同时进入首版

## 你需要给出的答复项

- `TASK-001` 是否通过并进入实施拆分
- 哪些设计保留，哪些降级，哪些延后
- 实现优先级排序
- 下一位主责任角色是谁

## 推荐写回格式

```md
## Producer 确认项
- ...

## Producer 答复项
- 保留：...
- 延后：...
- 不做：...

## 决策
- TASK-001：通过 / 有条件通过 / 退回修改
- 下一角色：...
```

## 当前默认下一棒

- 若评审通过：handoff 给 `web-tech-lead`，拆工程任务并开始实现规划
- 若评审未通过：handoff 给 `systems-economy`，先修设计文档
