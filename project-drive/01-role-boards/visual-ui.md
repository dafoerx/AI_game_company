# visual-ui role board

**角色**：视觉 / UI 负责人  
**当前阶段定位**：把系统文档翻译成清晰可点击、可扫读、可反馈的 Web 界面结构

## 开始前必读

1. `project-drive/00-active-context.md`
2. `project-drive/02-task-cards/in-progress/TASK-001_核心循环与资源流及首页信息架构.md`
3. `project-drive/04-meeting-notes/2026-03-26_TASK-001_系统设计评审请求.md`
4. `docs/02-MVP规划/MVP_SCOPE.md`

## 当前轮到你时要做什么

- 审查首页信息架构是否过密或过散
- 判断顶部状态栏、项目卡片、员工卡片、事件弹窗是否能快速理解
- 给出 Web 原型第一版最小组件结构
- 明确哪些信息必须首屏展示，哪些应该折叠到二级界面

## 当前确认项

- 现金 / 算力 / 士气 / 口碑的表达是否清晰
- 项目与员工信息是否适合卡片式展示
- 拖拽分配是否是首版最稳方案，还是先改为点击分配
- 警告态、低资源态、失败前提示是否足够明显

## 你需要给出的答复项

- 首页布局建议是否需要调整
- 关键信息卡片的最小字段集
- 推荐的组件优先级
- 需要系统策划删减或重排的界面内容

## 推荐写回格式

```md
## Visual/UI 确认项
- ...

## Visual/UI 答复项
- 首屏保留：...
- 首屏延后：...
- 交互建议：...
- 风险：...

## 下一角色
- prototype-qa / producer
```

## 当前默认下一棒

- 常规情况下 handoff 给 `prototype-qa`
- 若发现信息结构超出 MVP 承载，直接回给 `systems-economy`
