## 1. 核心循环设计

### 1.1 15～30分钟体验节奏

#### 前3分钟：首次决策
- 玩家创建公司，选择初始项目类型（小型游戏）
- 配置初始团队：1名真人骨干 + 2名AI员工
- 理解基础界面：现金、算力、士气、进度条
- 做出第一个决策：分配AI员工到研发任务

#### 前10分钟：第一次重要选择
- 完成第一个小项目，获得少量现金和口碑
- 面临选择：招聘更多AI员工 vs 升级现有员工能力
- 遭遇第一个挑战：士气下降或算力不足
- 学习使用"休息"或"购买算力"功能

#### 前20分钟：首次扩张成果
- 积累足够现金，可以开启中型项目
- 解锁第一个升级：AI算法优化或团队规模扩大
- 感受到明显的效率提升：研发速度加快
- 面临新的取舍：多元化项目组合 vs 专注深度

#### 前30分钟：MVP验证点
- 能够运行2-3个并行项目
- 开始感受到资源压力（现金、算力、士气平衡）
- 明确下一步扩张方向（更大的项目类型、更多的AI员工、更高级的工具链）
- 产生继续优化和扩张的动力

### 2.2 高风险点调整（基于评审反馈）

#### 士气系统调整（从惩罚性→管理性）
**原设计问题**：士气过低时效率惩罚过强，可能被视为惩罚性系统
**调整方案**：
1. **增加高士气奖励**：士气>80时，效率+20%
2. **降低低士气惩罚**：士气<30时，效率最低降至60%（原设计可能更低）
3. **增加恢复途径**：增加团建活动、加薪等正面恢复方式
4. **改善反馈机制**：明确显示士气变化原因和影响

#### 现金流失败调整（从突然死亡→逐步恶化）
**原设计问题**：现金为负3天直接倒闭，可能被视为突然死亡
**调整方案**：
1. **三级预警系统**：
   - 一级预警：现金储备不足30天开支（黄色警告）
   - 二级预警：现金储备不足15天开支（橙色警告）
   - 三级预警：现金储备不足7天开支（红色警告+紧急选项）
2. **挽救机制**：
   - 紧急贷款选项（高利息，-口碑）
   - 临时裁员降本（-士气但保留核心团队）
   - 项目紧急变现（提前结束项目获得部分收入）
3. **倒闭条件**：连续3天现金为负且未采取任何挽救措施

#### 信息过载调整（基于界面评审）
**原设计问题**：首屏信息密度过高，可能分散新手注意力
**调整方案**：
1. **采用单屏主界面+模态窗口**布局
2. **默认隐藏次要信息**，按需展开
3. **强化视觉层次**：资源状态>项目列表>员工列表>其他信息
4. **渐进式信息展示**：新手引导期间逐步介绍功能

### 2.3 验证指标调整（基于测试评审）

#### 核心验证指标（首版必须验证）
1. **前5分钟理解度**：
   - 操作成功率：完成项目分配的操作步骤≤3步
   - 资源识别率：≥80%玩家正确说出5种资源用途
   - 目标理解度：≥90%玩家清晰说出游戏目标

2. **10～15分钟策略感**：
   - 决策思考时间：首次重要决策前思考30-60秒
   - 资源取舍意识：≥70%玩家主动提及资源限制
   - 策略多样性：≥3种不同扩张路径被选择

3. **30分钟继续意愿**：
   - 主动继续率：≥60%玩家完成体验后主动选择继续
   - 扩张感知度：≥75%玩家明确感受到效率提升
   - 下一步计划：≥80%玩家有明确下一步目标

#### 验证不推荐的延期功能
即使技术可实现，但不利于首轮验证的功能：
1. **复杂拖拽分配**：延期，首版用点击分配替代
2. **多级导航**：延期，首版用单屏+模态窗口替代
3. **详细财务报表**：延期，首版只显示关键现金变化

## 3. 资源系统设计

### 3.1 核心资源定义（首版5个）

| 资源 | 类型 | 上限 | 获取方式 | 消耗方式 | 影响 | 风险 |
|------|------|------|----------|----------|------|------|
| **现金** | 货币 | 无（但现金告急会导致公司倒闭） | 项目收入、外部投资、紧急借贷 | 员工工资、算力购买、办公室租金、工具购买 | 公司生存基础，决定扩张速度 | 现金为负超过X天，公司倒闭 |
| **算力** | 消耗品 | 有上限，可升级 | 购买、免费配额、时间恢复 | AI员工执行任务、算法训练、数据处理 | AI员工效率，并行任务数量 | 算力不足时AI效率大幅下降 |
| **士气** | 状态值 | 0-100 | 项目成功、团队活动、加薪 | 加班、项目失败、随机负面事件 | 整体工作效率，员工流失率 | 士气过低导致员工离职、效率惩罚 |
| **口碑** | 累积值 | 无上限 | 项目成功、用户好评、媒体报道 | 项目失败、负面事件 | 吸引人才、获得投资机会、提高项目收入系数 | 口碑过低难以招聘优秀人才 |
| **研发进度** | 进度值 | 0-100% | AI员工工作、算法加成 | 无（只积累） | 项目完成度，决定发布时间 | 进度停滞导致机会成本 |

### 3.2 资源流转关系

```
现金 ──┬──> 支付工资 ──> 维持士气
       ├──> 购买算力 ──> 驱动AI工作
       ├──> 办公室租金 ──> 提供工位上限
       └──> 购买工具 ──> 提升效率系数

算力 ──> AI员工工作 ──> 研发进度 ──> 项目完成

研发进度 ──> 项目发布 ──┬──> 现金收入
                       ├──> 口碑积累
                       └──> 士气提升

士气 ──> 工作效率系数 ──> 实际工作产出

口碑 ──> 收入系数加成 ──> 现金收入增加
```

### 3.3 核心公式与关系（整合TASK-001A数值参数）

#### 3.3.1 工作产出公式（基于数值平衡表）
**实际工作产出 = 基础产出 × 士气系数 × 工具系数 × 匹配系数**

**参数定义**：
- **基础产出**：
  - AI员工L1：10进度/天
  - AI员工L2：15进度/天  
  - AI员工L3：20进度/天
  - 真人骨干：15进度/天（不消耗算力）
- **士气系数**（管理性设计，非惩罚性）：
  - 士气>80：效率+20%（系数1.2）
  - 士气60-80：正常效率（系数1.0）
  - 士气30-60：效率-20%（系数0.8）
  - 士气<30：效率-40%（系数0.6）⚠️ *原设计可能更低，已调整*
- **工具系数**：
  - 工具等级1：1.0
  - 工具等级2：1.2（升级成本$2,000）
  - 工具等级3：1.4（升级成本$5,000）
  - 工具等级4：1.6（升级成本$10,000）
- **匹配系数**：
  - 完全匹配（专业对口）：1.2
  - 部分匹配：1.0
  - 不匹配：0.8

#### 3.3.2 算力消耗公式
**AI员工算力消耗 = 任务基础消耗 × 员工等级**

**参数定义**：
- **任务基础消耗**（每小时）：
  - 小型项目：10算力/小时
  - 中型项目：25算力/小时
  - 大型项目：50算力/小时
- **员工等级乘数**：
  - AI员工L1：1.0×
  - AI员工L2：1.4×（7算力/小时小型项目）
  - AI员工L3：2.0×（10算力/小时小型项目）

#### 3.3.3 项目收入公式
**项目收入 = 基础收入 × 口碑系数 × 项目质量系数**

**参数定义**：
- **基础收入**：
  - 小型项目：$3,000
  - 中型项目：$8,000
  - 大型项目：$20,000
- **口碑系数**：1.0 + (口碑/1000)
  - 口碑0-500：系数1.0-1.5
  - 口碑500-2000：系数1.5-3.0
  - 口碑>2000：系数3.0+（上限5.0）
- **项目质量系数**（随机分布）：
  - 优秀：1.2×（概率20%）
  - 良好：1.0×（概率60%）
  - 普通：0.8×（概率20%）

#### 3.3.4 士气变化公式（管理性设计）
**士气变化 = 基础衰减 + 项目影响 + 管理决策 + 随机事件**

**参数定义**：
- **基础衰减**：-1/天（无项目时）
- **项目影响**：
  - 项目成功：+10~20（根据项目规模）
  - 项目失败：-15~25
- **管理决策**：
  - 加班工作：-5/天（当员工疲劳度>60）
  - 团建活动：+15（消耗$500现金）
  - 员工加薪：+10（增加10%工资成本）
- **随机事件影响**：±5~20（根据事件类型）

#### 3.3.5 现金流公式
**现金变化 = 项目收入 - 固定支出 - 算力购买 - 其他支出**

**参数定义**：
- **固定支出**（每月）：
  - 办公室租金：$1,000（等级1）
  - 员工工资：真人$1,000 + AI员工$500/人
- **算力购买**：$100 = 100算力（立即获得）
- **其他支出**：升级成本、团建活动等

#### 3.3.6 失败条件公式（逐步恶化设计）

**现金流断裂条件**：
1. **一级预警**：现金储备不足30天开支（黄色警告）
   - 条件：现金 < 月固定支出 × 1.0
   - 提示："现金流紧张，建议减少开支或增加收入"
2. **二级预警**：现金储备不足15天开支（橙色警告）
   - 条件：现金 < 月固定支出 × 0.5
   - 提示："现金流危险，考虑紧急措施"
3. **三级预警**：现金储备不足7天开支（红色警告）
   - 条件：现金 < 月固定支出 × 0.25
   - 提示："即将破产！必须立即采取行动"
4. **倒闭条件**：连续3天现金为负且未采取挽救措施

**士气崩溃条件**：
1. **低士气警告**：士气<40（效率降低）
2. **危险士气**：士气<20（员工可能离职）
3. **士气崩溃**：士气<10持续1周（核心员工离职）

### 5.3 来源与消耗（基于数值平衡表调整）

#### 现金（管理性现金流系统）
- **来源**：
  - 项目收入：小型$3,000，中型$8,000，大型$20,000
  - 外部投资：随机事件触发，+$5,000~$50,000
  - 紧急借贷：高利息选项，现金+$10,000，口碑-100
- **消耗**：
  - 员工工资：真人$1,000/月，AI员工$500~$1,200/月
  - 算力购买：$100 = 100算力（立即获得）
  - 办公室租金：$1,000~$3,000/月（根据等级）
  - 工具升级：$2,000~$10,000一次性投入
- **压力点与预警**（三级预警系统）：
  - **一级预警**：现金储备不足30天开支（黄色警告）
  - **二级预警**：现金储备不足15天开支（橙色警告）
  - **三级预警**：现金储备不足7天开支（红色警告+紧急选项）
  - **倒闭条件**：连续3天现金为负且未采取挽救措施

#### 算力（产能管理系统）
- **来源**：
  - 购买：$100 = 100算力（立即获得）
  - 时间恢复：10~30点/小时（根据服务器等级）
  - 免费配额：初始500，可升级至2000
- **消耗**：
  - AI员工工作：小型项目10算力/小时，中型25，大型50
  - 算法训练：升级AI员工时消耗
  - 数据处理：处理大型项目时额外消耗
- **压力点**：
  - 算力使用率>80%：效率开始下降
  - 算力不足：AI员工工作效率降至50%
  - 算力枯竭：AI员工停止工作

#### 士气（管理性士气系统）
- **来源**（正面激励）：
  - 项目成功：+10~20（根据项目规模）
  - 团建活动：+15（消耗$500现金）
  - 员工加薪：+10（增加10%工资成本）
  - 正面事件：随机+5~15
- **消耗**（管理性消耗）：
  - 加班工作：-5/天（当员工疲劳度>60）
  - 项目失败：-15~25（根据项目规模）
  - 负面事件：随机-5~20
  - 自然衰减：-1/天（无项目时）
- **影响机制**（非惩罚性设计）：
  - 士气>80：效率+20%（奖励性加成）
  - 士气60-80：正常效率
  - 士气30-60：效率-20%（管理性提醒）
  - 士气<30：效率-40%（需要干预）
  - 士气<20持续1周：核心员工可能离职

#### 口碑（累积影响系统）
- **来源**：
  - 项目成功：+50~200（根据项目质量和规模）
  - 用户好评：随机事件+100~500
  - 媒体报道：特殊事件+300~1000
- **消耗**：
  - 项目失败：-100~300
  - 负面事件：随机-50~200
  - 紧急借贷：-100（每次借贷）
- **影响**：
  - 收入系数：1.0 + 口碑/1000（上限5.0）
  - 招聘吸引力：口碑越高，越容易招聘高级AI员工
  - 投资机会：口碑>500解锁更多投资机会

### 5.6 失败风险（基于评审调整）

#### 现金流断裂（逐步恶化，非突然死亡）
1. **预警阶段**：
   - 一级预警：现金储备不足30天开支（黄色警告）
   - 二级预警：现金储备不足15天开支（橙色警告）
   - 三级预警：现金储备不足7天开支（红色警告+紧急选项）
2. **挽救机制**：
   - 紧急贷款：+$10,000现金，-100口碑，高利息
   - 临时裁员：降低月度支出，-20士气/人
   - 项目紧急变现：提前结束项目获得50%收入
3. **倒闭条件**：连续3天现金为负且未采取任何挽救措施

#### 士气崩溃（管理性提醒，非惩罚性）
1. **低士气影响**：
   - 士气<40：效率-20%，黄色警告
   - 士气<30：效率-40%，橙色警告
   - 士气<20：核心员工可能离职，红色警告
2. **恢复机制**：
   - 团建活动：+15士气，-$500现金
   - 员工加薪：+10士气，增加10%工资成本
   - 减少加班：降低工作强度，自然恢复
3. **崩溃条件**：士气<10持续1周且未采取恢复措施

#### 算力枯竭（产能限制）
1. **影响阶段**：
   - 算力使用率>80%：效率开始下降
   - 算力不足：AI员工工作效率降至50%
   - 算力枯竭：AI员工停止工作
2. **恢复方案**：
   - 购买算力：$100 = 100算力
   - 升级服务器：提高算力上限和恢复速度
   - 调整项目分配：减少高耗能项目

#### 口碑崩坏（累积影响）
1. **影响阶段**：
   - 口碑<0：难以招聘新员工，投资机会减少
   - 口碑<-100：项目收入系数降至0.5
   - 口碑<-500：可能触发公关危机事件
2. **恢复方案**：
   - 成功项目：高质量项目快速恢复口碑
   - 公关活动：消耗现金改善公众形象
   - 时间修复：随时间自然恢复（缓慢）

## 6. 交互流程设计（基于技术+界面评审）

### 6.1 项目分配流程（点击分配，非拖拽）
**简化交互流程**：
```
1. 点击员工卡片 → 显示"分配"按钮
2. 点击"分配"按钮 → 弹出项目选择模态窗口
3. 选择目标项目 → 显示确认对话框
4. 确认分配 → 更新员工和项目状态
```

**技术实现要点**：
- 员工卡片：显示当前状态和"分配"按钮
- 项目选择模态：显示可用项目列表（过滤已满项目）
- 确认对话框：显示分配后的影响（算力消耗变化等）
- 状态更新：实时更新员工和项目关联状态

### 6.2 时间推进流程（手动推进为主）
**交互流程**：
```
1. 点击"推进时间"按钮 → 显示时间选择器
2. 选择推进跨度（1天/1周/1月）→ 确认推进
3. 计算推进期间变化 → 显示变化摘要
4. 确认应用变化 → 更新游戏状态
```

**技术实现要点**：
- 时间选择器：预设跨度选项，支持自定义天数
- 变化计算：批量计算资源变化、项目进度、事件触发
- 变化摘要：可视化显示关键变化（现金、算力、士气等）
- 状态批量更新：一次性更新所有相关状态

### 6.3 事件处理流程（简化决策）
**简单事件**（Toast通知）：
- 自动显示，3秒后消失
- 不影响当前操作流程
- 可点击查看详情

**重要决策**（模态窗口）：
- 暂停游戏，强制处理
- 明确选项和后果
- 倒计时提醒（如有时间限制）

**关键选择**（简化选项）：
- 最多3个明确选项
- 每个选项有清晰后果描述
- 支持"稍后决定"（如有时间余量）
