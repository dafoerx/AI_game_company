## 2. 资源系统设计

### 2.1 核心资源定义（首版5个）

| 资源 | 类型 | 上限 | 获取方式 | 消耗方式 | 影响 | 风险 |
|------|------|------|----------|----------|------|------|
| **现金** | 货币 | 无（但现金告急会导致公司倒闭） | 项目收入、外部投资、紧急借贷 | 员工工资、算力购买、办公室租金、工具购买 | 公司生存基础，决定扩张速度 | 现金为负超过X天，公司倒闭 |
| **算力** | 消耗品 | 有上限，可升级 | 购买、免费配额、时间恢复 | AI员工执行任务、算法训练、数据处理 | AI员工效率，并行任务数量 | 算力不足时AI效率大幅下降 |
| **士气** | 状态值 | 0-100 | 项目成功、团队活动、加薪 | 加班、项目失败、随机负面事件 | 整体工作效率，员工流失率 | 士气过低导致员工离职、效率惩罚 |
| **口碑** | 累积值 | 无上限 | 项目成功、用户好评、媒体报道 | 项目失败、负面事件 | 吸引人才、获得投资机会、提高项目收入系数 | 口碑过低难以招聘优秀人才 |
| **研发进度** | 进度值 | 0-100% | AI员工工作、算法加成 | 无（只积累） | 项目完成度，决定发布时间 | 进度停滞导致机会成本 |

### 2.2 资源流转关系

```
现金 ──┬──> 支付工资 ──> 维持士气
       ├──> 购买算力 ──> 驱动AI工作
       ├──> 办公室租金 ──> 提供工位上限
       └──> 购买工具 ──> 提升效率系数

算力 ──> AI员工工作 ──> 研发进度 ──> 项目完成

研发进度 ──> 项目发布 ──┬──> 现金收入
                       ├──> 口碑积累
                       └──> 士气提升

士气 ──> 工作效率系数 ──> 实际工作产出

口碑 ──> 收入系数加成 ──> 现金收入增加
```

### 2.3 核心公式与关系

1. **实际工作产出 = 基础产出 × 士气系数 × 工具系数**
   - 士气系数：士气/100（0.5-1.2）
   - 工具系数：1.0起步，最高2.0

2. **AI员工算力消耗 = 任务难度 × 员工等级**
   - 小型项目：10算力/小时
   - 中型项目：25算力/小时
   - 大型项目：50算力/小时

3. **项目收入 = 基础收入 × 口碑系数 × 项目质量**
   - 口碑系数：1.0 + 口碑/1000
   - 项目质量：0.8-1.2随机

4. **士气变化 = 项目成功奖励 - 加班惩罚 - 随机事件影响**

## 4. 系统设计（结构化表）

### 4.1 系统目标
设计一套能够支撑15～30分钟持续体验的经营循环系统，让玩家在资源限制下做出有意义的选择，感受到扩张和成长的乐趣。

### 4.2 资源字段（数据库字段）
```json
{
  "cash": 10000,           // 现金，整数
  "compute_power": 500,    // 当前算力，整数
  "compute_max": 1000,     // 算力上限，整数
  "morale": 85,           // 士气，0-100整数
  "reputation": 1200,     // 口碑，整数
  "office_level": 1,      // 办公室等级，决定员工上限
  "server_level": 1       // 服务器等级，决定算力上限
}
```

### 4.3 来源与消耗

#### 现金
- **来源**：项目收入、外部投资、紧急借贷
- **消耗**：员工工资、算力购买、办公室租金、工具升级
- **压力点**：每月固定支出 > 现金储备时

#### 算力
- **来源**：购买（现金换算力）、时间恢复（每小时X点）
- **消耗**：AI员工工作、算法训练
- **压力点**：并行项目过多导致算力不足

#### 士气
- **来源**：项目成功、团建活动、加薪
- **消耗**：加班工作、项目失败、负面事件
- **压力点**：士气<30时员工效率大幅下降

### 4.4 核心公式/关系
1. **项目进度增长 = ∑(AI员工产出 × 士气系数 × 工具系数)**
2. **AI员工产出 = 员工等级 × 任务匹配度**
3. **现金变化 = 项目收入 - 固定支出 - 算力购买**
4. **士气变化 = 基础衰减 + 事件影响 + 管理决策**

### 4.5 成长节奏

#### 初期（0-10分钟）
- 目标：完成第一个小型项目
- 资源：现金$5k，算力500，士气80
- 限制：2名AI员工，1个并行项目
- 学习：基础操作、资源概念

#### 中期（10-20分钟）
- 目标：积累资源，开启中型项目
- 资源：现金$15k，算力800，士气波动
- 限制：需要升级办公室或服务器
- 学习：资源平衡、员工管理

#### 扩张期（20-30分钟）
- 目标：运行2个并行项目
- 资源：现金$25k，算力1200，士气稳定
- 解锁：更多项目类型、更高级AI员工
- 体验：感受到规模效应和扩张乐趣

### 4.6 失败风险
1. **现金流断裂**：连续3天现金为负 → 公司倒闭
2. **士气崩溃**：士气<20持续1周 → 核心员工离职
3. **算力枯竭**：算力为0 → AI员工停止工作
4. **口碑崩坏**：口碑<0 → 难以获得新项目

### 4.7 验证点
1. 玩家是否理解5种资源的作用？
2. 玩家是否会在资源间做出取舍？
3. 15分钟后玩家是否还想继续玩？
4. 扩张是否带来明显的乐趣提升？
