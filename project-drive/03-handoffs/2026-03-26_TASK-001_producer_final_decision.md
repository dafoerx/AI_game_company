## Producer 裁决项

### 1. 范围确认：5种核心资源+4种失败条件
**裁决**：✅ **批准保留**，复杂度在首版承载范围内

**理由**：
- 技术评审确认5种资源状态管理可行
- 数值细化提供了具体参数支撑
- 4种失败条件形成完整风险体系，但有足够预警机制
- 测试评审确认验证指标可观测

**调整**：
- 失败条件警告机制需前置实现（现金流预警、士气警报）
- 口碑系统可简化初期实现（只影响收入系数）

### 2. 节奏确认：15～30分钟体验阶段性目标
**裁决**：✅ **批准**，节奏合理，符合原型验证目标

**具体批准**：
- **前5分钟**：理解基本操作与资源概念 ✅
- **前15分钟**：完成第一个项目，面临首次决策 ✅  
- **前30分钟**：完成2-3个项目，进行首次升级 ✅

**调整**：
- 增加里程碑检测点：记录玩家实际达成时间
- 提供"快速测试模式"：可选时间加速，便于验证

### 3. 风险确认：设计中的高风险点
**裁决**：⚠️ **需前置调整3个高风险点**

**高风险点处理**：
1. **士气系统感知风险**：从"惩罚性"调整为"管理性"
   - 增加高士气奖励（效率+20%）
   - 降低低士气惩罚强度（效率最低降至60%）
   - 增加正面恢复途径（团建、加薪）

2. **现金流断裂风险**：从"突然死亡"调整为"逐步恶化"
   - 增加三级预警：30天→15天→7天储备不足
   - 提供紧急贷款选项（高利息，-口碑）
   - 允许临时裁员降本（-士气）

3. **信息过载风险**：简化首屏信息密度
   - 采纳visual-ui建议：单屏主界面+模态窗口
   - 默认隐藏次要信息，按需展示
   - 强化视觉层次和分组

### 4. 优先级确认：可降级/延后的验证指标
**裁决**：📋 **批准以下降级/延后项**

**首版降级实现**：
1. **拖拽分配** → **点击分配**（技术建议）
2. **复杂多级导航** → **单屏+模态窗口**（技术+界面建议）
3. **实时自动推进** → **手动推进为主**（技术建议）
4. **复杂事件决策** → **简化决策界面**（界面建议）

**验证指标降级**：
1. **详细财务分析** → 只追踪核心指标（现金变化趋势）
2. **多维度效率分析** → 只关注项目完成时间和收入
3. **长期留存数据** → 首版只验证30分钟体验

## 界面设计最终批准

### 批准visual-ui设计方案：
1. **布局**：单屏主界面（顶部状态栏+主工作区+底部操作栏）
2. **交互**：点击分配 → 项目选择模态窗口
3. **信息层级**：三级信息展示（始终可见→按需展开→二级界面）
4. **组件优先级**：按三个阶段实现（核心循环→体验完善→扩展功能）

### 调整要求：
1. **顶部状态栏**：现金、算力、士气必须始终可见，口碑可折叠
2. **项目卡片**：显示进度、规模、分配员工数、预计完成时间
3. **员工卡片**：显示类型、状态、分配项目、疲劳度（仅真人）
4. **事件处理**：重要决策用模态窗口，简单通知用Toast
