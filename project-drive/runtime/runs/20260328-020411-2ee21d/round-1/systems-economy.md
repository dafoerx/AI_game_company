### 1) 本角色判断（systems-economy）

1. **同意先冻结MVP变量，不扩机制。**  
   v1只保留：到店率、等待耐心、预算倾向、锅底必选点单、库存消耗、缺货惩罚、现金/满意度/利润/流失四项结果指标。口味偏好、随机事件、会员等全部放入v1.1。

2. **建议采用“离散回合+表驱动”保证可实现与可复现。**  
   - 单营业日：`12回合 × 30秒`（约6分钟）  
   - 到店：`arrivals_t = round(base_arrivals_t * arrival_factor)`  
   - `arrival_factor = clamp(0.8 + satisfaction_prev/200, 0.8, 1.3)`  
   - `base_arrivals_t` 用固定数组（便于QA复现）

3. **点单与消耗用最小SKU实现：锅底/肉/菜/饮。**  
   - 订单结构：`锅底1（必选）+ 肉x + 菜y + 饮z(可选)`  
   - 消耗：`consume_i = min(stock_i, order_i)`  
   - 缺货：锅底缺货=整单取消；其他缺货=删项并记满意度惩罚。

4. **收支模型建议同时保留“现金变化”和“经营利润”两条口径。**  
   - 现金：`cash_end = cash_start - purchase_cost + revenue - fixed_cost`  
   - 经营利润（复盘）：`profit_day = revenue - COGS(consumed) - waste_cost - fixed_cost`  
   这样既能反映资金链风险，也能看出浪费问题。

5. **满意度必须可追踪到因子，便于技术日志与QA定位。**  
   建议单客评分：  
   `sat_i = clamp(100 - 12*wait_turns - 15*broth_shortage - 8*other_shortage_cnt - 10*max(0, bill/budget-1), 0, 100)`  
   当日满意度：`sat_day = clamp(0.7*sat_prev + 0.3*avg(sat_i), 0, 100)`。
