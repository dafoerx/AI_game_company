### 1) 本角色判断（visual-ui）

1. **当前技术/规则方向正确，但UI口径尚未冻结，暂不宜直接判定可验收**。  
   现在缺的是“事件发生后，界面哪里变、怎么变、何时变”的统一定义。

2. **“实时更新且正确”在UI侧必须是“可见且可理解”**，不只是数值变了。  
   建议强制：`现金/满意度/库存`常驻同一位置，并在事件后给出增减高亮（颜色+符号）。

3. **信息架构应坚持低层级操作**：采购、营业、日结三阶段入口明确，核心动作不超过2层点击。  
   否则3个营业日连续游玩会出现“找不到入口”的软阻断。

4. **必须补齐“防误操作与异常状态UI”**：  
   现金不足、库存不足、顾客等待超时时，要有禁用态/警示态/可恢复提示，避免玩家卡死。

5. **本轮可接受占位视觉，不接受信息不一致**。  
   动效和美术可延后，但指标位置、文案、颜色语义（正负反馈）必须统一。
