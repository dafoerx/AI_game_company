### 1) 本角色判断（visual-ui）

1. **方向已基本一致，但还未达到“可交互验收”的完成态**。  
   目前对事件驱动与三指标实时更新已有共识，但 `TASK-004` 中缺少完整的 UI 契约落盘（事件触发后界面反馈规则）。

2. **“实时更新且正确”在UI侧必须满足“看得见 + 看得懂 + 不误解”**。  
   建议固定：顶部常驻 `现金 / 满意度 / 库存`，事件后提供短暂增减提示（`+/-数值` + 颜色语义），避免玩家只看到跳值不知道原因。

3. **信息架构应保持低认知负担**：采购、营业、日结三个阶段入口必须明确且不超过两层操作。  
   目标是支持玩家连续3个营业日无讲解完成，不出现“找不到下一步”的软阻断。

4. **异常态反馈是P0可玩性条件，不是UI装饰项**。  
   `现金不足/库存不足/状态非法` 必须有：按钮禁用态 + 原因提示 + 可恢复路径（例如“去采购”“返回营业”）。

5. **可接受占位视觉，不接受口径不一致**。  
   动效、美术细节可后置，但指标命名、颜色语义、按钮状态、提示文案等级（info/warn/error）本轮必须统一。
