### 1) 本角色判断（systems-economy）

1. **同意采用“0-100统一数值轴 + 线性可解释公式”**，满足 M1 可实现与可调试目标。  
   我建议状态字段固定为：`knotTags`、`trust`、`stress`、`attachment`、`adoptability`（5个主字段，不再扩维）。

2. **可领养度必须“分数 + 档位 + 原因”三层输出**，避免黑箱。  
   建议公式（每日结算一次，前端可实时读）：
   \[
   adoptabilityScore = 0.45*trust + 0.35*(100-stress) + 0.15*resolveRate + 0.05*attachBalance
   \]
   - `resolveRate = 已解决心结数/总心结数*100`
   - `attachBalance = 100 - min(100, |attachment-65|*2)`（依恋过低/过高都扣分，鼓励“能亲近也能放手”）

3. **送养判断采用“硬门槛 + 软分档”**（可解释且可测试）：  
   - 硬门槛：`trust < 25` 或 `stress > 80` => **不可送养**  
   - 软分档：  
     - `score >= 75` 且 `trust >= 60` 且 `stress <= 40` => **可送养**  
     - `50~74` => **观察中**  
     - `<50` => **不可送养**  
   - 前端展示 `reasons[]`（如“应激过高”“高风险心结未解”）。

4. **失败惩罚需要钳制，防止劝退**。  
   单次失败上限：`trust -8`、`stress +10`、`adoptability变化绝对值 <= 12`；并加入恢复机制：**连续2次正向互动触发“安抚加成”**（额外 `trust+3`,`stress-3`）。

5. **给出可落地的初始参数与事件规模**（满足验收）：  
   - 3类动物初始参数建议：  
     - 狗：`trust40 stress58 attachment50`（起步可领养度约36）  
     - 猫：`trust30 stress66 attachment38`（约28）  
     - 兔：`trust26 stress74 attachment34`（约23）  
   - 事件总量首版锁定 **9个**（每类3个，模板可复用）。
