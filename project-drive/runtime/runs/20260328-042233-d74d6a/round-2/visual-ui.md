### 1) 本角色判断（visual-ui）

1. **TASK-004 的信息架构已可收敛为“单流程闭环”**：编辑卡片 → 预览校对 → 发布入池 → 匹配列表 → 匹配详情，尽量不跳出当前任务上下文。  
2. **“可解释匹配”在 UI 上必须固定为四块**：`匹配分`、`成功率`、`正向因子(≥3)`、`风险因子(≥3+温和提示)`，否则玩家无法形成可决策认知。  
3. **验收标准可视化需要状态机落地**：每只动物必须能看到且仅看到一条卡片的状态流转 `草稿 -> 已发布 -> 已入推荐池`。  
4. **视觉语气必须保持“治愈但不回避风险”**：风险文案要给可行动建议，避免责备式表达（如“建议先提升稳定陪伴时段再尝试匹配”）。  
5. **当前仍不满足“跨角色最终共识”条件**：`systems-economy / web-tech-lead / prototype-qa` 本轮未提交有效结论，UI 不能单方面签收终版。
