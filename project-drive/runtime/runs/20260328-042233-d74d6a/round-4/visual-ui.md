### 1) 本角色判断（visual-ui）

1. **TASK-004 的 UI 可以按 MVP 落地，但必须坚持“单闭环、低认知负担”**：建议固定为  
   `编辑卡片 → 预览 → 发布 → 入推荐池 → 匹配列表 → 匹配详情`，不再增加分支页。  
2. **“1宠1卡”需要强可见约束**：在卡片页显示唯一状态标签（草稿/已发布/已入池），若重复生成必须弹出“覆盖草稿”确认，避免用户误以为可多卡并行。  
3. **匹配结果必须“先结论、再解释”**：详情页首屏只放四块核心信息：`匹配分 / 成功率 / 正向因子 / 风险因子`；`scoreBreakdown` 放在“为什么是这个分数”折叠层，减少首屏拥挤。  
4. **风险提示要给行动建议，不只给警告**：例如“陪伴时长不足”后附“建议先进行2次晚间互动再尝试匹配”，否则用户会有无力感。  
5. **可读性标准要前置**：因子标签限制为“正向最多2条 + 风险最多2条”，其余收纳“查看更多”；移动端需保持一屏可看到匹配结论与主按钮。

**冲突点 / 待确认**

- `recommendationLevel` 的中文分级文案与情绪语气未冻结（“不建议”是否过重，是否改为“暂缓匹配”）。
