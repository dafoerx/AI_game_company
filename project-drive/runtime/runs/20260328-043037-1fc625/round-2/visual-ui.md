1) **本角色判断（visual-ui）**

1. **当前 TASK-005 不能直接按“已完成”推进，需先修订验收口径**。  
   现有标准偏产量（4界面/12资源/2动效），缺少 UI 质量门槛（信息层级是否清晰、关键操作是否可见、状态反馈是否易读）。

2. **四个核心界面必须先统一信息架构，再做视觉深化**。  
   我建议每页都坚持“1个主目标 + 2个次信息层”：  
   - 动物档案：先看状态，再看背景；  
   - 互动界面：先看可选行动，再看预期影响；  
   - 送养确认页：先看推荐结论，再展开成功率构成；  
   - 回访照片墙：先看情绪回访结论，再看细节内容。  

3. **送养成功率展示应“可解释但不拥挤”**。  
   和 systems-economy 一致要拆解因子，但 UI 上不应一次塞满公式；建议默认显示“总评+3个关键因子”，其余用“展开详情”查看，避免信息密度过高。

4. **12张情绪资源必须与状态系统一一映射，并保持跨页一致视觉语义**。  
   不能只在头像里换表情，还要同步状态标签/色条/微动效提示，否则玩家会感到“演出好看但决策无依据”。

5. **两段动效要服务情绪节奏，不得干扰操作连续性**。  
   必须支持：可跳过、不可叠播、失败降级静态帧；并保证按钮与关键文案在动效期间仍可见或可快速恢复。
