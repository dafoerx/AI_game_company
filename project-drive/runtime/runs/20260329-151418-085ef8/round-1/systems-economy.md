### 1) 本角色判断（systems-economy）

1. **10-15分钟“可感知价值”路径可行，但必须压缩单只动物前半节奏**  
   建议首轮体验拆为：  
   - 0-3分钟：接收动物 + 读线索 + 教学  
   - 3-9分钟：2-3个照护决策（观察/陪伴/小游戏）+ 第1段心结  
   - 9-13分钟：达成“可匹配领养”阈值 + 家庭匹配  
   - 13-15分钟：送养反馈（照片卡+文字）  
   数值上建议：前4个“游戏日”内可完成首次闭环，避免首轮流失。

2. **《MVP范围清单》最小内容量（建议锁死）**  
   - 动物：**3只**（狗/猫/兔各1，创伤原型各异）  
   - 家庭：**4组模板家庭**（偏好+禁忌可覆盖基础匹配）  
   - 事件：**10个**（每只动物2个关键心结=6；通用随机事件4）  
   - 小游戏：**1个**（可复用一套机制做动物差异参数）  
   这组量级能支持“首轮完整体验 + 二轮有轻微变化”。

3. **过场规格要“轻演出、重反馈”**  
   MVP建议：  
   - 心结节点：静态插画/立绘切换 + 文本（40-80字）  
   - 送养节点：照片墙3张 + 1条回访卡文案  
   - **不做真实视频生产**（可保留“伪动态”占位，不进入首版硬依赖）  
   这样可显著降低美术与前端整合成本。

4. **不做项需要再补一条“系统级不做”**  
   除现有不做项外，建议新增：  
   - 首版不做“多结局分支树”和“长期跨月追踪关系网”  
   否则事件与状态组合会指数膨胀，压垮M2周期。

5. **里程碑与减配策略必须绑定触发条件**  
   - 若M2第2周事件完成率 <80%：随机事件4减到2（保关键剧情）  
   - 若M2第3周性能不达标：过场动效降级为纯静态卡片  
   - 若M3前BUG高优>10条：家庭模板4减到3（保匹配闭环）
